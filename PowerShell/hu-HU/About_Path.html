Path adatelérési utak

Azokat a parancsmagokat, amelyekkel kifejezetten a PowerShell-hez kifejlesztett, hierarchikus felépítésű PSProvider adattárhelyeken tárolt adatelemeket kezeljük, a PowerShell alapparancsainak (core commands ) nevezzük. A PSProvider adattárhelyeken tárolt adatelemeket egyedi azonosítóját adatelérési útnak (Path ) hívjuk.

Egy adatelérési út lehet abszolút vagy relatív. Egy abszolút adatelérési út a hierarchikus (faszerkezetű) adattárhely gyökerénél kezdődik, amit PSDrive meghajtónak nevezünk. Minden meghajtónak egyedi megnevezése van. Az abszolút adatelérési út a meghajtótól az egymásba ágyazott tárhelyeken (Location ) keresztül vezet egészen a tárolt adatelemig (Item), és annak a nevével végződik. A meghajtó elé tetszés szerint feltüntethető az adattárhely neve is, tehát egy abszolút elérési út teljes szerkezete a következőképpen írható le: Path = [<PSProvider>::][<PSDrive>:]\[<Location>\…]<Item> Abszolút elérési utat láthatunk például a parancssor elejét jelző ún. prompt  karakterláncban.

Egy relatív elérési út egy adott tárhelytől indul, és a hierarchiában fel-le közlekedve ér el az azonosított adatelemig. A relatív elérési út leírásában egy pont az aktuális tárhelyet, két pont pedig a hierarchiában eggyel magasabb szintet jelöli. Egy A tárhelyről a vele szomszédos B tárhelyre tehát a ../B relatív útvonallal hivatkozhatunk.
