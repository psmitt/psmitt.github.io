Path adatelérési utak

Azokat a parancsmagokat, amelyekkel kifejezetten a PowerShell-hez kifejlesztett, hierarchikus felépítésű PSProvider adattárhelyeken tárolt adatelemeket kezeljük, a PowerShell alapparancsainak (core commands ) nevezzük. A PSProvider adattárhelyeken tárolt adatelemeket egyedi azonosítóját adatelérési útnak (Path ) hívjuk.

Egy adatelérési út lehet abszolút vagy relatív. Egy abszolút adatelérési út a hierarchikus (faszerkezetű) adattárhely gyökerénél kezdődik, amit meghajtónak (PSDrive ) nevezünk. Minden meghajtónak egyedi megnevezése van. Az abszolút adatelérési út a meghajtótól az egymásba ágyazott tárhelyeken (Location ) keresztül vezet egészen a tárolt adatelemig (Item), és annak a nevével végződik. A meghajtó elé tetszés szerint feltüntethető az adattárhely neve is, tehát egy abszolút elérési út teljes szerkezete a következőképpen írható le: Path = [<PSProvider>::][<PSDrive>:]\[<Location>\…]<Item> Abszolút elérési utat láthatunk például a parancssor elejét jelző ún. prompt  karakterláncban.

Egy relatív elérési út egy adott tárhelytől indul, és a hierarchiában fel-le közlekedve ér el az azonosított adatelemig. A relatív elérési út leírásában . az aktuális tárhelyet, .. pedig a hierarchiában eggyel magasabb szintet jelöli. Egy A tárhelyről a vele szomszédos B tárhelyre tehát a ../B relatív útvonallal hivatkozhatunk. Relatív elérési utak abszolút alakjának meghatározására szolgál a Convert-Path parancsmag.

Adatelérési utak kezdő- és végszakaszát a Join-Path parancsmaggal összeilleszthetjük még akkor is, ha az útszakaszok leírásában a * helyettesítő karaktert használjuk: a -Resolve kapcsoló felderíti számunkra az összes létező adatelérési utat, amely a megadott kezdő- és végszakaszra illeszkedik (ld. még Test-Path, Resolve-Path).

A Split-Path parancsmag egy adatelérési út meghatározott részét adja eredményül aszerint, hogy melyiket adjuk meg az alábbi kapcsolók közül:
  • -Qualifier a meghajtó nevét kettősponttal zárva;
  • -NoQualifier az elérési utat a meghajtó neve nélkül;
  • -Parent az elérési út által meghatározott adatelem tárhelyét;
  • -LeafBase az elérési út által meghatározott adatelem nevét;
  • -Extension az elérési út által meghatározott adatelem kiterjesztését;
  • -Leaf az elérési út által meghatározott adatelem nevét és kiterjesztését.
