Hogyan tartsuk naprakészen a PowerShell parancsértelmezőt?

A PowerShell telepített verziójának megállapítása

Az adott platformon telepített PowerShell verziószámát a parancsértelmező elindítása nélkül is megállapíthatjuk a pwsh -version paranccsal. Amennyiben a PowerShell előzetes (preview ) verzióját (is) telepítettük a számítógépre, annak verziószámát a pwsh-preview -version paranccsal jeleníthetjük meg.

A PowerShell új kiadásainak figyelése

A https://github.com/PowerShell/PowerShell/releases/latest webcím mindig az utolsó stabil verzió kiadási oldalára mutat, amelynek alján szintén megtaláljuk a különböző platformokra előkészített telepítő csomagokat. Ha értesülni szeretnénk az újabb kiadásokról, állítsuk be a kiadások figyelését a https://github.com/PowerShell/PowerShell/releases oldalon.

A PowerShell újabb verziójának telepítése

Az önállóan telepített PowerShell verziófrissítését az újabb verzió önálló telepítésével hajtsuk végre (ld. A PowerShell telepítése). A PowerShell segítségével a böngésző használata nélkül is letölthetjük és elindíthatjuk a telepítő programot a következő PowerShell utasítással: Invoke-Expression "& { $(Invoke-RestMethod -Uri https://aka.ms/install-powershell.ps1) } -UseMSI -Preview -AddToPath" Az utasítás kiadása előtt természetesen el kell indítani a PowerShell parancsértelmezőt a konzolon a pswh vagy pwsh-preview paranccsal.

A PowerShell verziófrissítése csomagkezelő használatával

A .NET globális eszközként telepített PowerShell programcsomag valamennyi platformon egységesen a dotnet tool update --global PowerShell utasítással frissíthető a parancssorból. Érdekességképpen megjegyezzük, hogy ugyanez az utasítás akkor is feltelepíti a PowerShell legújabb verzióját, ha a PowerShell előtte egyáltalán nem is volt a számítógépre telepítve. Amennyiben a számítógépen már az elérhető legújabb verzió fut, akkor mindössze üzenetet kapunk erről, és a frissítés értelemszerűen nem történik meg.
